Developer Documentation

Overview

Welcome to Elite's developer docs! This section provides a comprehensive overview of our web application's codebase and how to contribute to it. PHP is the main programming language used on our hotel booking website; JavaScript, HTML, and CSS are also used. We make use of the Bootstrap framework to improve styling and guarantee cross-device responsiveness.

Getting Started

Please take the following actions to configure Elite locally using XAMPP:

  1. Use git clone [your repository URL] to clone the repository.

  2. Transfer the project folder to the htdocs directory of XAMPP.

  3. Launch XAMPP and make sure MySQL and Apache are up and working.

  4. 4. In phpMyAdmin, create a new database.

  5. Update the config.php file with your database credentials to configure the database.

  6. Open your browser and go to https://github.com/MahaAlzoubah/Elite

Developer Guidelines



Welcome to the community supporting Elite development! The purpose of these guidelines is to facilitate your effective participation in the project. Before you begin working on Elite, please review these a bit.

Developer Communication

The documents and guidelines of the community must be followed by every member.

It is recommended that developers connect via Microsoft Teams or email. These are not hard regulations, but rather guidelines. They seek to enhance collaboration, optimise our process, and provide advantages to our goods as well as to individual developers.

The Elite development team is available for assistance if you have any issues regarding these criteria.

Code Format

  1. There should only be one code command per line.
  2. Keep the space between code blocks at two lines.

Code Scope

  1. Classes that have just default constructor values should use default constructors.
  2. Use the keyword "final" unless it pertains to constants.

Other Code-Related Information

  1. Each variable's name should have meant and be pertinent to its intended use.
  2. There should be a brief statement describing the purpose of each group or block of code.

Creating Secure Software

Although it might not be the most thrilling aspect of writing code, it is imperative to take security threats into account. Analyzing security risks should start early in the project and continue all the way through to conclusion.

  1. Learn about the security threats related to the project's components.
  2. Steers clear of depending just on default configurations and software that comes with the box.

Which Language Should I Use for Programming?

Regarding programming languages, we don't have any hard regulations. We use JS, CSS, PHP, and HTML now, however we're open to changes for better project outcomes.

Code Comments

  1. Comments on the code should provide an explanation for its actions.
  2. Make sure your indentation is standardized.
  3. Make use of the horizontal whitespace (code blocks and paragraphs).
  4. Don't write lines that are longer than 75-80 characters.

Using the bug Tracker

The recommended method for reporting issues, asking for features, and making pull requests is the bug tracker.

Requests for Features

  1. Make sure the objectives and scope of the project align with your idea.
  2. Make a strong argument to argue in favor of the feature.


Requests for Pulls
  1. Good pull requests include patches, upgrades, and new features.
  2. Aim for focused pull requests and steer clear of irrelevant commits.
  3. Before you begin, find out the scope of the pull request from a maintainer.

Bug Report

Contains:

Title: Concise and precise. Summary: In-depth justification is provided. Visual evidence: Video or screenshots are provided. Expected versus actual result: succinct and direct. Steps that led to the issue are those that are repeatable. Environment: OS, browser, screen resolution, and zoom degree. Log from the console: Find the source of the issue. URL of the source: To rapidly determine the problem. Priority and severity: The degree of impact and the urgency of the investigation. Name of the reporter, recipient, deadline, and exchange with the customer or user. Create a report using this data and send it to elite.github@gmail.com for support.
